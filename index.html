How do you set up at HTML File?

In order to let the web browser know we are using HTML we must start out document with a document type declaration that looks like this:
<!DOCTYPE html>
This declaration provides the browser with 2 pieces of information:
  1. The type of document
  2. the HTML version to expect
The declaration doesn't actuallly add any HTML structure or content.

This must be the first line of code in your HTML document. Currently, HTML5 is the most up to date version, but in the future once a new standard has overtaken version 5, beginning with the declaration will make sure that your document is forever interpreted correctly. Lastly, HTML code always saves in a file with a .html extension. After you write the declaration, then anything between the <html> and </html> will be interpreted as HTML code.

The Head Element

The head element goes above the body element of an html document. It contains metador for a web page. Metadata is info about the page that isn't displayed directly on the web page.
A browser's tab displays the title specified in the title tag. The title tag is always inside of the head tag.

The Anchor Tag

You can add links to another web page by adding the anchor element and including the text of the link between the opening and closing tags.
An anchor tag must have and href attribute (hyperlink reference) Here's and example:

<a href="https://www.wikipedia.org/">This Is A Link To Wikipedia</a>

would display as this: This Is A Link to Wikipedia

Another attribute of the anchor tag in the target attribute. The target attribute specifies how a link should open. target="_blank" to open the link in a new window. Leaving out the target attribute will open the link in the same window.

Here is the same link with a target attribute: This Is A Link to Wikipedia

When wanting to link to other pages in a static website:

Usually web developers store HTML files in the root directory or a main folder where all the files for a project are stored. For example let's say that we had this file structure:

project-folder/
  • about.html
  • contact.html
  • index.html

We can link web pages together using a relative path (shows the path to a local file or a file on the same website. An absolute path is a full url

Let's say that we are on the index.html and we want to go to the contact page.

<a href="./contact.html">Contact</a> (would show up as contact underlined as a link)

The ./ tells the browser to look for the file in the current folder

Images as a link

You can turn an image into a link by wrapping the image tag in anchor tags

What are some tools that developers can use to make code easier to interpret?

  1. Whitespace
    -Each element should have it's own line
  2. Indentation
    -The World Wide Web Consortium (W3C) recommends 2 spaces of indentation when writing HTML for easy visualization of elements nested within other elements. Remember that the position of elements in a browser is independent of the amount of whitespace or indentation in the index.html file.

Commenting in code:

  1. Helps you and others to understand your code (especially if you decide to come back and review it at a much later date).
  2. Allow you to experiment with new code without having to delete old code.
